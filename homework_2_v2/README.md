# 啟明 QiMing — 中文程式語言編譯器與虛擬機

## 專案概述

「啟明 (QiMing)」是一門現代化的正體中文程式語言，使用 C++17 從零實作完整的編譯器前端（詞法分析、語法分析）、直譯器、位元組碼編譯器、以及堆疊機虛擬機（Stack VM），適用於教學與基礎演算法實作。

## 語言規格

### 語言定位
現代化、泛用型的正體中文程式語言，語法結構清晰，適合教學與基礎演算法實作。

### 型態系統
強型態 (Strongly Typed)、靜態型別 (Statically Typed)。

| 中文關鍵字 | 對應型別 | 說明 |
|-----------|---------|------|
| `整數` | int | 64 位元整數 |
| `小數` | float | 64 位元浮點數 |
| `字串` | string | UTF-8 文字字串 |
| `布林` | bool | `真` / `假` |

### 關鍵字一覽

| 中文關鍵字 | 用途 | 對應英文 |
|-----------|------|---------|
| `整數` | 整數型別宣告 | int |
| `小數` | 浮點數型別宣告 | float |
| `字串` | 字串型別宣告 | string |
| `布林` | 布林型別宣告 | bool |
| `如果` | 條件分支 | if |
| `否則` | 條件分支（否則分支） | else |
| `當` | 迴圈 | while |
| `回傳` | 函式返回值 | return |
| `印出` | 輸出 | print |
| `真` | 布林真值 | true |
| `假` | 布林假值 | false |

### 支援的運算子

| 類別 | 運算子 |
|------|--------|
| 算術 | `+`, `-`, `*`, `/`, `%` |
| 比較 | `==`, `!=`, `<`, `<=`, `>`, `>=` |
| 邏輯 | `!` (NOT) |
| 賦值 | `=` |
| 一元 | `-` (負號) |

### 範例程式

#### 費氏數列（遞迴 + 迴圈）

```text
整數 費氏數列(整數 項數) {
    如果 (項數 <= 1) {
        回傳 項數;
    } 否則 {
        回傳 費氏數列(項數 - 1) + 費氏數列(項數 - 2);
    }
}

整數 總和 = 0;
整數 計數 = 1;
當 (計數 <= 10) {
    總和 = 總和 + 計數;
    計數 = 計數 + 1;
}

印出("1加到10的總和為: ", 總和);
印出("費氏數列第10項為: ", 費氏數列(10));
```

#### 階乘（遞迴）

```text
整數 階乘(整數 n) {
    如果 (n <= 1) {
        回傳 1;
    } 否則 {
        回傳 n * 階乘(n - 1);
    }
}
印出("10! = ", 階乘(10));
```

#### 質數判斷

```text
布林 是質數(整數 n) {
    如果 (n <= 1) { 回傳 假; }
    整數 i = 2;
    當 (i < n) {
        如果 (n % i == 0) { 回傳 假; }
        i = i + 1;
    }
    回傳 真;
}
```

## 系統架構

```
原始碼 (.qm)
     │
     ▼
┌──────────┐
│  Lexer   │ ─── Token 串流
│ 詞法分析  │
└──────────┘
     │
     ▼
┌──────────┐
│  Parser  │ ─── 抽象語法樹 (AST)
│ 語法分析  │
└──────────┘
     │
     ├──────────────────────┐
     ▼                      ▼
┌──────────────┐     ┌──────────────┐
│  Interpreter │     │  Compiler    │
│  AST 直譯器   │     │  位元組碼編譯  │
└──────────────┘     └──────┬───────┘
                            ▼
                      ┌──────────────┐
                      │   Stack VM   │
                      │  堆疊機虛擬機  │
                      │  + GC        │
                      └──────────────┘
```

### 執行模式

1. **直譯器模式（預設）** — 解析成 AST 後直接走訪執行
2. **位元組碼 VM 模式**（`--vm` 旗標）— 將 AST 編譯為堆疊機位元組碼，在自製虛擬機中執行

## 建置與執行

### 編譯

```bash
cd qiming
make
```

或手動編譯：

```bash
g++ -std=c++17 -Wall -Wextra -O2 -Isrc -o qiming src/*.cpp
```

### 執行

```bash
./qiming tests/fib.qm            # 直譯器模式（預設）
./qiming --vm tests/fib.qm       # 位元組碼 VM 模式
```

### 清理

```bash
make clean
```

### 跑全部測試

```bash
for f in tests/*.qm; do echo "=== $f ==="; ./qiming "$f"; echo; done
```

## BNF 語法

啟明語言的正式語法定義如下：

```bnf
<Program>      ::= <Statement>*
<Statement>    ::= <VarDecl> | <FuncDecl> | <IfStmt>
                 | <WhileStmt> | <ReturnStmt> | <PrintStmt> | <ExprStmt>
<VarDecl>      ::= <Type> <Identifier> "=" <Expression> ";"
<FuncDecl>     ::= <Type> <Identifier> "(" <ParamList>? ")" "{" <Statement>* "}"
<ParamList>    ::= <Type> <Identifier> ("," <Type> <Identifier>)*
<IfStmt>       ::= "如果" "(" <Expression> ")" "{"
                   <Statement>* "}"
                   ( "否則" "{" <Statement>* "}" )?
<WhileStmt>    ::= "當" "(" <Expression> ")" "{" <Statement>* "}"
<ReturnStmt>   ::= "回傳" <Expression> ";"
<PrintStmt>    ::= "印出" "(" <ArgList> ")" ";"
<Assignment>   ::= <Identifier> "=" <Assignment> | <Comparison>
<Comparison>   ::= <Term> (("+" | "-" | "==" | "!=" | "<" | "<=" | ">" | ">=") <Term>)*
<Term>         ::= <Unary> (("*" | "/" | "%") <Unary>)*
<Unary>        ::= ("-" | "!") <Unary> | <Primary>
<Primary>      ::= <Integer> | <Float> | <String> | <Bool>
                 | <Identifier> ("(" <ArgList>? ")")?
                 | "(" <Expression> ")"
<FuncCall>     ::= <Identifier> "(" <ArgList>? ")"
<Type>         ::= "整數" | "小數" | "字串" | "布林"
```

## 記憶體管理

虛擬機實作了基礎的「標記-清除 (Mark-and-Sweep)」垃圾蒐集器 (GC) 管理字串記憶體：

1. **配置**：字串常數透過 `GC::allocateString()` 配置，以鏈結串列追蹤所有物件
2. **標記 (Mark)**：GC 觸發時掃描值堆疊與全域變數表，所有可觸及的字串物件被標記
3. **清除 (Sweep)**：走訪物件鏈結串列，釋放未標記的物件
4. **閥值**：每配置 100 個字串物件觸發一次 GC

## 各階段實作細節

### 階段一：詞法分析器 (Lexer)
- **檔案**：`lexer.h`, `lexer.cpp`, `token.h`, `token.cpp`
- 完整的 UTF-8 中文字元處理（三/四位元組序列辨識）
- 支援所有關鍵字（整數、小數、字串、布林、如果、否則、當、回傳、印出、真、假）
- 跳脫字元處理（`\n`, `\t`, `\"`, `\\`）
- 單行註解 `//` 與多行註解 `/* */`
- 雙字元運算子（`==`, `!=`, `<=`, `>=`）

### 階段二：語法分析器與 AST (Parser)
- **檔案**：`parser.h`, `parser.cpp`, `ast.h`, `ast.cpp`
- 14 種 AST 節點類別（7 種陳述式 + 7 種運算式）
- 遞迴下降解析器，支援完整的運算子優先權
- Visitor 設計模式（`StmtVisitor` / `ExprVisitor`）
- 回溯 (backtracking) 處理賦值與函式呼叫的歧義

### 階段三：直譯器 (Interpreter)
- **檔案**：`interpreter.h`, `interpreter.cpp`
- Visitor Pattern 走訪 AST 並直接執行
- 巢狀作用域環境 (`Environment`)，支援變數遮蔽
- 函式呼叫透過 `ReturnException` 機制傳遞回傳值
- 型別自動轉換（整數 ↔ 小數混和運算）

### 階段四：位元組碼編譯器 (Compiler)
- **檔案**：`compiler.h`, `compiler.cpp`, `bytecode.h`
- 18 個 OpCode 指令集
- 統一常數池 (`Constant` 結構，含型別標記)
- 兩階段編譯：先收集所有函式與全域變數，再編譯本體
- 區域變數索引追蹤（呼叫 frame 內索引定址）
- 條件/迴圈跳躍位址先行佔位 (`emitPlaceholder`) 後回填 (`emitAt`)

### 階段五：虛擬機與 GC (Stack VM & GC)
- **檔案**：`vm.h`, `vm.cpp`
- 堆疊機：值堆疊 + 呼叫 frame 堆疊
- Frame 結構含 `code` 指標、`ip`、`stackBase`、`returnIP`
- 函式呼叫時切換 frame，返回時回復
- Mark-and-Sweep GC：可追蹤的字串物件鏈結串列
- GC 閥值：100 個字串物件後自動觸發回收
